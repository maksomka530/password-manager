# Simple Python Password Manager

> *«Храните пароли в безопасности, не полагаясь на сторонние сервисы.»*

Лёгкий менеджер паролей с графическим интерфейсом на Python и Tkinter. Позволяет хранить учётные данные локально в зашифрованном виде.

## 🔐 Возможности

- **Мастер-пароль** — единый ключ доступа к хранилищу.
- **Шифрование AES-128 (Fernet)** — надёжная защита данных.
- **Генератор случайных паролей** — создание стойких комбинаций символов.
- **Удобная таблица** — просмотр всех записей (сервис, логин, пароль).
- **Копирование пароля в буфер обмена** — быстрая вставка в формы авторизации.
- **Автоматическое сохранение** — изменения фиксируются мгновенно.
- **Портативность** — все файлы хранятся в одной папке в профиле пользователя, легко перенести или создать резервную копию.
- **Кроссплатформенность** — работает на Windows, Linux, macOS при наличии Python.

## 📦 Установка и запуск

### 1. Установите Python 3
Скачайте с [python.org](https://www.python.org/downloads/) и установите, отметив пункт «Add Python to PATH».

### 2. Установите зависимости
Откройте терминал (PowerShell, CMD) и выполните:
pip install cryptography pyperclip
### 3. Скачайте скрипт
Сохраните файл password_manager.py в удобную папку.

### 4. Запустите
bash
python password_manager.py
При первом запуске программа создаст папку PasswordVault в домашнем каталоге пользователя и предложит задать мастер-пароль. Запомните его — восстановление доступа к данным без мастер-пароля невозможно.

## 🧠 Использование
Добавление записи: нажмите «Добавить» → введите сервис, логин и пароль (можно сгенерировать новый).

Редактирование: выберите строку в таблице → «Изменить».

Удаление: выберите строку → «Удалить».

Копирование пароля: выберите строку → «Копировать пароль».

Генерация пароля: кнопка «Сгенерировать» — пароль появится на экране и автоматически скопируется в буфер обмена.

Все изменения сразу сохраняются в зашифрованный файл vault.dat.

### 📁 Структура хранилища
text
PasswordVault/
├── vault.key          # Хеш мастер-пароля (только для проверки, не содержит секретов)
└── vault.dat          # Зашифрованное хранилище (AES-128)
vault.key используется для проверки правильности ввода мастер-пароля.

vault.dat — само хранилище. Без мастер-пароля расшифровать его практически невозможно.

### 🛡️ Безопасность
Используется библиотека cryptography с алгоритмом Fernet (AES-128 в режиме CBC + HMAC-SHA256).

Ключ шифрования генерируется из мастер-пароля через SHA-256 (для повышения стойкости можно заменить на PBKDF2).

Пароли никогда не хранятся в открытом виде.

Потеря мастер-пароля ведёт к необратимой потере всех данных.

 ### 🐛 Известные особенности
При вводе мастер-пароля символы не отображаются (стандартное поведение).

Размер окна можно изменять вручную, таблица адаптируется.

Для копирования пароля используется системный буфер обмена — рекомендуется очищать его после использования (в планах автоматизация).
